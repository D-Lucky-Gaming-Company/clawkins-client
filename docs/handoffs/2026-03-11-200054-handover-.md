# Feature Handover - Left Facing + Encounter Battle Prototype

## Scope Delivered

This handover covers the following tasks:

1. Character now visually faces left while moving left (mirrored frame fallback).
2. Encounter logic + battle UI prototype implemented (using `GdxGame` flow as reference pattern).
3. Encounter detection tightened to the player’s feet within the encounter tile (no broad rectangle triggers).

## 2) Encounter Logic + Battle UI Prototype

The structure follows the reference intent from `GdxGame`:

- world encounter detection
- transition into battle state
- simple turn loop with UI actions

### Runtime flow

1. Player moves so that their **feet point** (bottom center of the sprite) enters an encounter trigger tile.
2. `EncounterDetectionSystem` publishes `START_ENCOUNTER` once on overlap-enter.
3. `BattleService` starts a `BattleContext` using custom properties from that encounter object.
4. Exploration systems pause (`PlayerInputSystem`, `MoveSystem`, `EncounterDetectionSystem`).
5. `BattleOverlay` displays battle UI and handles inputs.
6. Player actions resolve through `BattleStateMachine`.
7. On victory/defeat/escape, pressing an interaction key (`Z`/`SPACE`/`ENTER`) closes the session and returns to exploration.

### Current battle controls

- **During player turn**:
  - `1` / `2` / `3`: Use Skill 1/2/3
  - `Z` / `SPACE` / `ENTER`: Quick-use Skill 1
  - `R`: Escape
- **After battle is finished** (`VICTORY` / `DEFEAT` / `ESCAPE`):
  - `Z` / `SPACE` / `ENTER`: Close finished battle and return to exploration

### Systems paused during active battle session

## 3) Battle State Machine Details (Prototype Level)

### Implemented

- Deterministic turn loop:
  - Player turn → enemy turn → player turn → …
- Damage formula:
  - Player: `max(1, skill.power - enemy.defense)`
  - Enemy: `max(1, enemy.attack - player.defense)`
- Player has 3 skills in the prototype, all defined from encounter object custom properties:
  - Skill 1: `skill1Name`, `skill1Power`
  - Skill 2: `skill2Name`, `skill2Power`
  - Skill 3: `skill3Name`, `skill3Power`
- Enemy and player combat stats are loaded from encounter object custom properties:
  - Enemy: `enemyHp`, `enemyAttack`, `enemyDefense`, `enemySpeed`
  - Player: `playerHp`, `playerAttack`, `playerDefense`, `playerSpeed`
- If any property is missing, defaults are used automatically.
- MP is removed from the current combat data model (`BattleUnit` now uses HP/ATK/DEF/SPD only).
- End states:
  - `VICTORY`, `DEFEAT`, `ESCAPE`
- Battle log string for UI display:
  - Stores human-readable last event (hit/defeat/escape).

### Not yet implemented

- Items/defend behavior still not complete.
- Encounter table lookups from JSON.
- Dedicated battle scene visuals/animations/audio.
- Reward application and persistence.
